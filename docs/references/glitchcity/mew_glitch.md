The Trainer escape glitch, also known as the Trainer-Fly glitch, is a glitch in Pokémon Red, Green, Blue (/wiki/Pok%C3%A9mon_Red_and_Blue), and Yellow (/wiki/Pok%C3%A9mon_Yellow) that allows the player to "escape" from a Trainer who has seen the player, before the Trainer battle starts. This puts the game in a glitched state, often resulting in an instant encounter, i.e., the player encountering a Pokémon or Trainer immediately upon returning to the map which the player escaped from. This encounter can be controlled since it is based on the last Special stat of the opponent in memory, so this glitch is often used to encounter and capture rare Pokémon, especially Mew (https://bulbapedia.bulbagarden.net/wiki/Mew) which cannot be obtained via either legitimate in-game methods or the simpler old man glitch (/wiki/Old_man_glitch). Established procedures to obtain Mew with this glitch are often known as the Mew trick (/wiki/Guides:Mew_trick) or the Mew glitch, and these names are sometimes also used to refer to this glitch in general.

In Japan, the glitch is often referred to as fifth's method (Japanese: fifth法), named after "fifthヽ(´ー｀)ノ◆Fi3PJTZKLQ" who found it and posted about it on 2ch. It has also been referred to as Special encounter glitch (Japanese: とくしゅエンカウント).

## Overview
## 
The name "Trainer escape glitch" usually refers to a two-part procedure: The player first escapes from a Trainer with Fly, Teleport, Dig, or an Escape Rope, then exploits the resulting glitched game state. For a basic example of the full procedure, see Mew trick (/wiki/Guides:Mew_trick).

There are multiple methods to perform a Trainer escape:

- The player can escape from a long-range Trainer by pressing Start mid-step. A long-range Trainer is a Trainer who can be forced one tile off-screen, and who will walk up to the player and challenge them to a battle when they appear.

- The player can exploit the Rival's effect (/wiki/Rival%27s_effect) caused by certain glitch items to allow the player to escape from not just long-range Trainers.

- The player can exploit a death-warp (#Death-warp), in which they receive a wild encounter in the grass or in a cave within a Trainer's vision and black out.
The first two methods have a side effect of disabling the Start menu and certain other actions, such as talking to an NPC.

The main effect of performing a Trainer escape is that the game "remembers" that a Trainer battle is about to happen in the map which the player escaped from. This is achieved by changing the meta-map script (/wiki/Meta-map_script) ID for that map from 0 to 1. When the player returns to that map, the game will try to perform the following sequence of events:

- It checks the last stored text box ID and uses it to display a textbox.

- A battle is triggered.

- Once the battle finishes, the game performs a final action depending on the type of battle.

- If the last battle was a wild pokémon encounter, it attempts to remove an NPC's sprite from the current map.

- If the last battle was a trainer encounter, it attempts to mark a trainer as defeated.
Finally, the game will revert the meta-map script (/wiki/Meta-map_script) ID for the current map back to 0, returning the game state back to normal.

## Effects and applications
## 

## Escaping the Trainer
## 
When a trainer spots the player, the game performs a number of actions:

- The meta-map script (/wiki/Meta-map_script) ID is changed to signify that a battle is expected to start.

- The trainer class is loaded into 0xCD2D, the team number is loaded into 0xCD2E.

- Bit 0 of 0xCD60 is set when the Trainer spots you, disabling the use of the Start menu and preventing you from talking to NPCs.

- Bit 0 of 0xD730 is set when the Trainer starts walking towards you. The game will not start the battle until this bit is reset, indicating that the NPC has stopped walking towards you.

## Reenabling the Start menu
## 
Bit 0 of 0xCD60 is usually cleared through three different methods:

- It is automatically reset when the player saves and reloads. By traveling to a nearby pokémon center, the player can use a PC to forcibly save the game by changing the current active box, allowing them to reset the game.

- It is automatically reset when the player wins a trainer battle, which can be done by engaging any trainer on a different map.

- It is automatically reset when the player blacks out, which can be done by losing any battle or by fainting the last remaining party member using poison.

## Resetting bit 0 of 0xD730
## 
After you've been spotted by a Trainer at range, the game expects the Trainer to walk up to you before talking to you. By escaping the Trainer, this animation never has the opportunity to occur, causing a few side effects.

If the player returns to the route they escaped the Trainer from, the encounter will never resume until this flag is cleared. Due to the altered meta-map script (/wiki/Meta-map_script) ID, Trainer sightlines will be disabled, though trainer encounters can still be initiated by talking directly to them.

If the player travels to a different map and walks into the sightline of a trainer that does not need to walk up to the player, the game will softlock if this flag is still set. This is because the game expects a walking animation to finish, which will never happen since the Trainer never starts a walking animation to begin with.

Bit 0 of D730 can be reset by getting any NPC to finish a walking animation. Common methods include:

- Moving a boulder that is pushed by Strength. Since Strength boulders are NPCs, the act of moving a boulder will automatically clear this flag once the boulder stops moving.

- Having a trainer on a different map walk up to you.

- Having an NPC finish a scripted walk animation, such as beating the Elite 4 and letting Prof. Oak finish a walking animation.

## Trainerless method
## 
Main article: Guides:Trainer escape glitch Trainerless method (/wiki/Guides:Trainer_escape_glitch_Trainerless_method)
This method can be pulled off without any Trainer at all and on routes without long-range Trainers (as long as at least one Trainer is present on the map). It also doesn't require a Pokémon with Fly, Teleport, Dig, or an Escape Rope.

## Opening a text box
## 
Once the player returns to the map they originally escaped from, while bit 0 of 0xD730 has been reset, the game will immediately attempt to open a text box. The contents of this text box are determined by the last stored text box ID in address 0xCF13.

If the player opened the Start menu right before entering the route, this text box ID value will be set to 0x00. This is interpreted as a special value, which will cause the game to open the Start menu unprompted.

## Text box ID matching
## 
If the player instead talked to an NPC or an interactable object (such as a TV), the game will re-use the most recently used text box ID value. Each map has their own unique lists of possible text box ID values, meaning that some maps may use ID values that are not used in other maps.

- If the last used text box ID value correspond to a valid text box, the game will play that text box normally. If the text box happens to belong to a Trainer, the text box may try to start that Trainer battle instead, messing up the meta-map script further. This may cause glitch meta-map script activation (#Glitch_meta-map_script_activation), with many possible glitchy effects including repeated Trainer battles and a "walking lag" effect.

- If the last used text box ID value does not correspond to a valid text box, the game will be forced to print text from a garbage location, which could potentially be located in RAM. This will often result in the game printing garbled and glitched text.
If the equivalent text box brought up an undefeated Trainer, it may cause the walking lag glitch (/wiki/Walking_lag_glitch) (see the article for steps).

## Viridian Forest arbitrary text box
## 
Exclusively to Yellow, the Town Map in the house with Speary the Spearow (predefined text box 0x41) will load an arbitrary text box from C331 in RAM. This is the x-position of OAM entry 13 (decimal) (FE31). OAM entry 13 will vary, sometimes depending on which NPCs are visible on the screen (if not all NPCs are present), it will correspond to the x-position of the left-half of the note on the wall about Speary instead. It is possible to manipulate C331 to values which are not actually on the screen. i.e. when Speary walks left off the screen, the game may load a very high x-position for it into memory.

In speedruns, this is manipulated to be 0xFE, because the starting 0xFE is a text command to open a Poké Mart; which can then be used for further glitches as part of the process to warp to the Hall of Fame (such as buffer overflow of the mart due to late 0xFF byte or the use of glitch items such as 4F (/wiki/ItemDex/Y:089)). The tile number of the NPC corresponds with the number of items in the Mart, and the rest of the OAM data which follow (including for entries 14 and onward) control individual items which appear in the Mart.

## Sea Route 21 arbitrary text box
## 
Main article: Sea Route 21 0x44 text box glitch (/wiki/Sea_Route_21_0x44_text_box_glitch)
Loading the hex:44 text box on Route 21 (via the shelves of Pokémon goods in Cinnabar Poké Mart) executes arbitrary text code from D2C3 in WRAM (the fifth character of the second Pokémon's nickname). This can be manipulated to run arbitrary code; for example with Super Glitch (/wiki/Super_Glitch) and the expanded party (/wiki/Expanded_party) one can convert items in the inventory into Pokémon nicknames and abuse this to obtain Mew as a gift Pokémon via the 08 text function (run ASM following the 08). This trick was documented by Torchickens.

## Glitch meta-map script activation
## 
Main article: Glitch meta-map script activation (/wiki/Glitch_meta-map_script_activation)
Glitch meta-map script activation often causes walking lag, with possible random sound effects being played or freezes (depending on your coordinates) when you return to the route you escaped from a Trainer (after following the finite or infinite Special stat encounter glitch). 
It may be caused by text box matching dialog for an undefeated Trainer and defeating them, or the following method:

1. Perform the 'fly away' part of the Trainer escape glitch. 

2. Change boxes and reset the game.

3. Return to the original route where the player flew away and defeat any Trainer.

Ultimately it can also be used as a setup for arbitrary code execution (/wiki/Arbitrary_code_execution).

## Trainer Fly Encounters
## 
When the player returns to the map they escaped from, the game will try to initiate that battle, causing an "instant encounter". However, many temporary variables pertaining to that battle would have been overwritten, which causes this battle to become quite different from the Trainer battle that is supposed to happen in the first place:

- If the player does any battle before returning to the route, then the Trainer class (0xCD2D) will be overwritten by the Special stat of the enemy Pokémon. Since most fixed wild Pokémon encounters (Legendary birds, Mewtwo, etc.) use the same code path as Trainer battles, the value of this Special stat determines whether the encounter is a Trainer battle or a wild Pokémon, and in most cases (Special modulo 256 = 1–199) it will be a wild Pokémon.

- Each Trainer class has a number of different rosters, and the roster number (0xCD2E) will be overwritten by the Attack stage of the enemy Pokémon, which is stored as an integer from 1 to 13 (-6 stages → 1, +6 stages → 13). If encountering a wild Pokémon, this number will instead become the level of the wild Pokémon.
Other methods also exist that can alter the contents of the encounter:

- Talking to any NPC that can trade a pokémon will cause the game to temporarily store the trade pokémon's nickname in memory. Because of this, the trainer class and roster will be overwritten by the 5th and 6th character of this nickname. This occurs even if the player has already previously performed a trade with the NPC. This does not work in the Japanese versions.

- Performing a trade with any NPC will cause the game to temporarily store the name "TRAINER" in memory. Because of this, the trainer class and roster will be overwritten by the 5th and 6th character of this name. This does not work in the Japanese versions.

- The player may be able to modify the trainer class and roster with Brock Fly (/wiki/Brock_through_walls#Brock_Fly).

## After the Trainer Fly encounter
## 
After the player wins or escapes from the battle, the game will try to set the defeat flag for the Trainer, but due to some subtle details[1] (#cite_note-1) in finding the address of the flag bit, may set an unrelated flag instead.

If the encounter was with a wild Pokémon, the game will also try to remove that wild Pokémon from the map altogether. A common example of this effect is removing Snorlax (#Removing_Snorlax).

- ↑ (#cite_ref-1) Maps with encounters usually stores the first Trainer header pointer in wTrainerHeaderPtr before executing the meta-map script (https://github.com/pret/pokered/blob/e3af20b907fad59bcbee7d36ab4912d0cc5f2935/home/trainers.asm#L12), so calling ReadTrainerHeaderInfo with a=2 (https://github.com/pret/pokered/blob/e3af20b907fad59bcbee7d36ab4912d0cc5f2935/home/trainers.asm#L197) will read the byte of the flag from this Trainer header, but the bit index (stored in wTrainerHeaderFlagBit) is from the last Trainer header actually loaded. This is not a problem in normal gameplay because Trainers on the same map always shares the same defeat flag byte (which means that the bit index may be greater than 7 for maps with 8 or more Trainers).
